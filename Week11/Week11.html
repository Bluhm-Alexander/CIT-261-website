Notes on javascript events and Transition and Animation Events

Standard JavaScript Event Handlers cover a wide variety of events. Here is a link for the page I will be referencing at length for the entirety of this page Link. There are a few ways you can handle events with JavaScript. For this page I'm mainly going to be handling events from the HTML page itself. The HTML page listens for events that the user causes and then immediately executes JavaScript code in response to the event.

For Problem1 I employed the onload function in the javascript itself because event handlers can be hard coded into the Javascript file itself. This method is called bubbling and capturing. This method was employed in the 90s and is not really used today. The standard syntax for hard coding event Listeners: button[i].addEventListener("click", buttonClick, capture). Like I said this method is old an unnecessary.

This all being said there are a couple other things to be said about event listeners. HTML5 has listeners that are specific to mobile devices. An example would be the Touch Events. onTouchStart is called when the user touches the screen, likewise onTouchEnd is called when the user's finger stops touching the screen.
